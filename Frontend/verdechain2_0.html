System Architecture

Built for automation,
designed for scale

A four-layer Python microservices stack processing raw freight data into compliance certainty in real time.

Layer 1
Ingestion
CSV Parser
pandas · polars
Normalizes shipment manifests, deduplicates on voyage ID hash
JSON Ingestor
FastAPI · asyncio
Consumes real-time GPS/telematics, validates via Pydantic
PDF OCR Engine
pdfplumber · pytesseract
Extracts invoice fields from scanned PDFs automatically
Normalizer
Schema Registry
Unifies all sources to canonical FreightRecord schema
Layer 2
Analytics
Carbon Calculator
GLEC Framework v3
Computes tCO₂e per route using mode-specific factors
Budget Tracker
TimescaleDB
Aggregates emissions against regulatory limits per route
STL Decomposer
statsmodels
Removes trend/seasonality to isolate anomalous spikes
Forecaster
scikit-learn
30-day rolling carbon prediction with confidence intervals
Layer 3
Optimization
Modal Shift Engine
Weighted scoring
Generates feasible modal substitutions per route
Pareto Ranker
70/30 weighting
Scores: carbon savings ×0.7 + cost impact ×0.3
Auto-Rebalancer
Rule engine
Applies top-ranked substitution at Critical threshold
Audit Logger
Immutable log
Records every decision for ESG reporting and audit
Layer 4
Alerting
Alert Classifier
4-level severity
Advisory / Warning / Critical / Emergency
Notification Router
SMTP · Webhook · SMS
Routes alerts to compliance team and external agencies
Dashboard WS
FastAPI WebSocket
Pushes live updates with sub-2s latency
ESG Reporter
Jinja2 → PDF
Auto-generates regulatory ESG reports on schedule
